intro

This page is my attempt at a tutorial for absolute beginners. I've tried to make it as simple and accessible as possible.

your first site

Lets make a website!

Websites usually have 3 parts:

  • HTML — This is where the words and pictures go.
  • CSS — This is where the styles and colors go.
  • Javascript — Lets you run code on people's browsers, which means you can do all sorts of fancy stuff.
For now, we'll just focus on HTML. Learn the basics before trying fancy stuff.

First, we need somewhere to write our HTML. Make a text file called index.html and paste the following code into it. 

<!DOCTYPE html>
<html>
    <head>
        <title>This is the title of my website.</title>
    </head>
    <body>
        <p>This is the content of my website.</p>
    </body>
</html>

Notice the indentation. You don't have to indent your code — HTML doesn't care. But I highly recommend it. If you don't, your code will look like a mess. It will get harder and harder to read as you add more stuff.

In HTML, anything surrounded by angle brackets is a "tag". Like <this>. Tags almost always come in pairs. The first one is the "opening tag". The second one is a closing tag, which has a slash (/) in it, like </this>. When you put them together, you get <this></this>.

Stuff can go between the opening and closing tags. For example: <p>This is a paragraph.</p>.

Different tags have different meanings, and do different things. You don't need to learn them all, you just need to know the common ones. Let's go through the ones in the code you just pasted:

  • <!DOCTYPE html> — This one isn't actually a tag. Notice how it doesn't have a closing tag. It's just something you have to put at the top of every .html file. You're gonna have to get used to weird exceptions like this. It's a common thing across HTML, CSS, and Javascript. Yes, it sucks. No, it's not gonna get better any time soon.
  • <html> — Everything (except the above !DOCTYPE) goes inside of this. By inside, I mean between the opening and closing tags.
  • <head> — This is where meta-information about the .html file goes, like the title of your page, what fonts to load, and what CSS and Javascript files to load. For now, we'll be keeping things simple.
  • <title> — This is the name of the page.
  • <body> — This is where the actual content goes, like paragraphs and images.
  • <p> — This is a paragraph.

You should be able to open the .html file in your browser. On Windows, you can right click index.html, click "Open with", and select your browser. If this doesn't work, You can put the file's absolute path in your browser's URL box. For example: C:/Users/dawn/Documents/website/index.html. You might need to replace backslashes (\) with forward-slashes (/) for that to work.

Whenever you make a change to the .html file, reload the page on your browser and it should update.

We're basically done! Go ahead and play around with it.

  • Try changing the page's title.
  • Try changing text in the paragraph.
  • Try adding new paragraphs.
  • Try putting a paragraph inside of a paragraph! It doesn't make much sense, but it should work.
  • Try removing a closing tag. It should still work, but your browser might get confused.

semantics and content

HTML is supposed to be accessible. Blind people use the internet too! They use something called a "screen reader". It's a program that looks at the HTML of your site and tries to verbally explain what is going on to the user. But websites are confusing - how does the screen reader tell what the different parts are? This is where semantics come in.

"Semantic HTML" just means using the right tags for each type of content. If you have a header, it should go in a <header> tag. If you have a footer, it should go in a <header> tag. If you have a section, put it in a <section> tag.

Here are some tags and their semantics:

  • <p> — This is a paragraph.
  • <h1> — This is a title.
  • <h2> — This is a subtitle.
  • <h3> — This is a sub-subtitle. There's also <h4>, <h5>, and <h6>. You get the idea.
  • <header> — This is for introductory content: navigation links, the title, your logo, etc.
  • <main> — This is for the main content of your page. Everything you're reading right now is inside it.
  • <footer> — This is usually for copyright info.
  • <nav> — This is for navigation links. Usually, it's inside the <header>.
  • <ul> — This is for an unordered list, like the one you're looking at right now.
  • <ol> — This is for an ordered list, which usually has numbers instead of bullet points.
  • <li> — This is an item inside a list. Usually, there are multiple of these inside a <ul> or <ol>. Each bullet point in this list is one of these.
  • <section> — This is section. Each chapter of this guide is in one of these.
  • <article> — Similar to <section>, but specifically for articles.

Here's a simplified version of this site: 

<!DOCTYPE html>
<html>
    <head>
        <title>Making a Website</title>
    </head>

    <body>
        <header>
            <nav>
                <ul>
                    <li><a href="index.html">home</a></li>
                    <li><a href="about.html">about</a></li>
                    <li><a href="media.html">media</a></li>
                    <li><a href="links.html">links</a></li>
                </ul>
            </nav>
        </header>

        <main>
            <h1>website tutorial.</h1>

            <section>
                <h2>table of contents</h2>
                <ul>
                    <li>intro</li>
                    <li>your first site</li>
                </ul>
            </section>
            
            <section>
                <h2>intro</h2>
                <p>blah blah blah</p>
                <p>blah blah blah</p>
                <p>blah blah blah</p>
            </section>
            
            <section>
                <h2>your first site</h2>
                <p>blah blah blah</p>
                <p>blah blah blah</p>
                <p>blah blah blah</p>
            </section>
        </main>

        <footer>
            copyright 2022 your mom
        </footer>
    </body>
</html>

The semantic definitions for these tags are not super exact, so sometimes it's not clear which tag you should use or where you should put something. That's fine. Just try your best, it doesn't have to be perfect.

Notice how I use blank lines to separate parts of my code. Your code will be easier to read if related things are grouped together and unrelated things are separated.

There's also the <div> tag. It has no semantic meaning. You'll see it used a lot, because people are too lazy to use the right tags. Don't do this, it's rude to blind people. Try to only use them when no tags exist for that meaning, or when there's no meaning at all. For example: Later on, we'll cover CSS, which often use divs to help layouts.

You don't have to use semantic HTML. You could just use divs for everything. You could manually style your divs to make them look like titles, lists, menus, buttons, whatever. But please don't.

By default, browsers have styles for some tags. This is so sites that have no CSS are still readable. Later we'll see how to override these styles.

links and images

Links use the <a> tag. They look like <a href="https://neocities.org">Neocities</a>. The href="" part is an attribute of the <a> tag. Inside the quotes goes the url of the link. The content of the <a> tag (the stuff in between the opening and closing tags) is the actual text of the link. So this example ends up looking like: Neocities.

You can link to other pages on your site this way. Usually, people put the main navigation links in a <nav> tag.

Images use the <img> tag. They look like <img src="tutorial_image.png">. Notice how this is one of the few tags that doesn't have an opening/closing pair! It's just one tag. The src attribute needs a path to the image file. If I put the image in a folder called images, my link would look like <img src="images/tutorial_image.png">. Here's what it looks like:

Figures use the <figure> tag. Anything inside it is part of the figure. Usually, it's an image. You can also put a <figcaption> tag inside it to give it a caption. Note that the default browser style might make the layout a bit wonky - we'll learn how to override/reset it later. Here's the result: 

<figure>
    <img src="tutorial_image.png">
    <figcaption>the tutorial image</figcaption>
</figure>
